# 배당주 종합 점수 웹앱 — 설계서 (SPEC)

> 이 문서는 **Claude Code에 그대로 넘기는 빌드 지시서**입니다.
> 핵심 로직(점수 엔진)과 작동하는 스캐폴드는 이미 구현되어 있습니다. Claude Code는 이걸
> 확장/배포하면 됩니다. **점수 산정 공식은 절대 임의로 바꾸지 마세요** — 사용자와 합의된 값입니다.

---

## 0. 한 줄 요약
사용자의 엑셀(`투자지침판`)을 재현한 웹앱. 종목 리스트를 **실시간 크롤링**으로 갱신하고,
**카테고리별 멀티팩터 점수**를 코드로 계산하며, **Gemini가 점수 해설**을 생성하고,
**수익 계산기**를 제공한다. 종목 추가/삭제 가능.

## 1. 아키텍처 (왜 이 구조인가)

```
[브라우저 프론트엔드]  ──HTTP──>  [Node/Express 백엔드]  ──>  네이버 금융 (크롤링)
   점수표/모달/계산기                  점수계산(코드) ──────>  Gemini API (해설만)
```

- **크롤링은 반드시 백엔드에서.** 브라우저 직접 크롤링은 CORS로 불가능하고 약관 위반 소지.
- **점수는 백엔드 코드(`scoring.js`)가 계산.** 정확·무료·즉시. Gemini에 점수를 맡기지 않는다.
- **Gemini는 "해설/정성평가"만.** 종목 클릭 시에만 호출 → 토큰 비용 통제.
- **새로고침(↻)** 누를 때만 크롤링 재실행 → 불필요한 요청/비용 방지.

## 2. 디렉토리

```
dividend-app/
├─ backend/
│  ├─ server.js      Express API (CRUD, refresh, explain, settings)
│  ├─ scoring.js     ★점수 엔진 (절대 공식 변경 금지)
│  ├─ crawler.js     네이버 금융 크롤러 (셀렉터는 보강 필요)
│  ├─ llm.js         Gemini 해설 생성
│  ├─ seed.json      엑셀에서 추출한 81개 종목 초기 데이터
│  ├─ scored_seed.json  점수까지 미리 계산된 버전 (프론트 데모용)
│  ├─ data.json      런타임 저장소 (최초 실행 시 seed로 자동 생성)
│  └─ package.json
└─ frontend/
   └─ index.html     단일 파일 데모 (점수표+모달+계산기, 인라인 시드로 단독 작동)
```

## 3. 점수 산정 로직 (★사용자 합의 — 변경 금지)

### 3-1. 2층 구조
```
종합점수 = max(0,  (1층 가중합) × (거시 섹터 승수)  −  (레드플래그 10점/건))
```

### 3-2. 1층: 카테고리 내 상대평가 (사분위 백분위)
각 지표를 **같은 카테고리 종목들 안에서의 백분위**로 환산(0~100).
**모든 가중치/승수/임계값은 `scoring.js`의 `DEFAULT_CONFIG`에 있고, 런타임 주입 가능.**
사용자가 설정탭에서 바꾼 값이나 AI 추천값을 `scoreStocks(stocks, config)`로 그대로 넘긴다.

| 요인 | 기본 가중치 | 구성 | 화면 우선순위 |
|---|---|---|---|
| 밸류 | 32% | PBR 60% + PER 40% | ★핵심 (초록 강조) |
| 수익성 | 23% | ROE 50% + (ROE/PBR) 50% | ★핵심 |
| 배당매력 | 18% | 현재 배당률 백분위 | ★핵심 |
| 배당지속성 | 17% | 전년배당 ÷ 4년평균배당 | ★핵심 |
| (보조) 배당성장 | 6% | 전년 > 4년평균이면 가점 | 보조 (회색) |
| (보조) 유동성 | 4% | 시총 로그스케일 | 보조 |

**핵심 4지표 합산 ≥ 80%** 유지가 원칙(화면·AI 추천 모두 이 우선순위 반영).
카테고리별 가중치 오버라이드는 `CATEGORY_WEIGHTS`. 가중치 합은 코드가 자동 정규화.

### 3-3. 2층: 거시 섹터 승수 (기준일 2026-05, `SECTOR_MULTIPLIERS`)
근거: 한은 기준금리 2.50% 동결 장기화, 초고환율(~1,510원), 여전채 4%+, 저성장(1.7%).
은행 ×1.06, 보험 ×1.05, 에너지·상사 ×1.04, 통신 ×1.02, 지주 ×1.00,
철강·증권 ×0.98, 내수·카드·기타금융 ×0.96. **개별종목이 아닌 카테고리 단위로만 적용.**
→ 거시 환경이 바뀌면 이 표만 갱신. `PUT /api/settings`로 런타임 조정도 가능.

### 3-4. 레드플래그 (각 −10점)
- 적자/왜곡: PER<0 또는 ROE<0 또는 배당성향<0 (분모 음수로 지표 무의미)
- 배당성향 100% 초과 (지속 불가능 배당)
- 배당 삭감: 전년배당 < 4년평균 × 0.7

## 4. API 명세 (구현 완료, server.js)

| 메서드 | 경로 | 설명 |
|---|---|---|
| GET | `/api/stocks` | 종목+점수 (내림차순) |
| POST | `/api/stocks` | 추가 `{name, code, category}` |
| DELETE | `/api/stocks/:code` | 삭제 (code 또는 name) |
| POST | `/api/refresh` | 크롤링→점수 재계산 (↻ 버튼) |
| POST | `/api/explain/:code` | Gemini 해설 (클릭 시, 캐시됨) |
| GET/PUT | `/api/settings` | 가중치·승수 조회/수정 (설정탭 슬라이더) |
| GET | `/api/config/default` | 기본 설정값 (초기화 버튼) |
| POST | `/api/recommend-config` | **AI 추천**: 오늘 거시·뉴스 기반 파라미터 조정안 (근거 포함) |

### 4-1. AI 추천 동작 (`llm.js`의 `recommendConfig`)
- Gemini가 **google_search 그라운딩**으로 오늘의 금리·환율·유가·업종 뉴스를 검색.
- **값을 창작하지 않음**: 현재 config를 출발점으로, "무엇을 왜 얼마나" 바꿀지 제안.
- 가드레일(프롬프트로 강제): 섹터승수 0.90~1.10·1회 ±0.06 이내, 핵심 4지표 ≥80% 유지.
- 반환 JSON: `{ asOf, macroSummary, recommendedConfig, changes:[{path,from,to,reason}], warnings }`.
- 프론트가 changes·근거를 보여주고, 사용자가 "적용"을 누르면 `PUT /api/settings` → 점수 재계산.

### 4-2. 초보자 가이드
- 프론트 `GLOSSARY` 객체에 13개 용어(PER/PBR/ROE/배당성향/배당률 + 6개 점수 항목 + 거시승수)의
  쉬운 설명·"어떻게 볼까" 팁 내장. 지표 옆 `?` 클릭 시 팝업.
- 핵심 지표는 초록색, 보조 지표는 회색으로 시각적 우선순위 구분.

## 5. Claude Code 할 일 (우선순위 순)

1. **크롤러 셀렉터 보강** — `crawler.js`의 네이버 파싱은 기본 골격만 있음.
   PER/PBR/ROE는 메인페이지, **배당수익률·배당성향·시가총액**은 종목별로 위치가 달라
   실제 페이지 구조(`/item/main`, `/item/coinfo`)를 보고 셀렉터를 정교화해야 함.
   → 안정성이 중요하면 **KIS Developers(한국투자증권) 무료 API**로 교체 권장.
   `fetchMetrics(code)` 시그니처만 유지하면 됨.
2. **종목코드 매핑** — `seed.json` 종목들에 6자리 `code` 채우기(현재 null). 크롤링의 키.
3. **프론트 React 전환(선택)** — 현재 `index.html`은 완전 작동하는 데모. 그대로 써도 되고,
   Vite+React로 옮기면 종목 추가/삭제 UI, 설정 패널을 더 깔끔히 만들 수 있음.
4. **.env 설정** — `GEMINI_API_KEY` 추가. `.env.example` 참고.
5. **배포** — 프론트는 정적 호스팅, 백엔드는 Render/Railway/Fly 등. CORS 허용 도메인 조정.

## 6. 수익 계산기 (구현 완료, index.html `simulate()`/`calc()`/`drawChart()`)
입력: **초기 원금 + 월 적립금**(콤마 자동), 기간, 주가상승률, 배당률, 배당성장률,
배당주기(연1/반기/분기/월), 배당소득세(15.4%) 토글. **과거 가정 / 미래 예측 모드** 전환.
가정 기반 시뮬(실제 주가 백테스트는 크롤러 연동 후 별도 구현). 월 단위 적립 + 배당 재투자 복리.
출력: 핵심 카드(평가금액/원금/누적배당/손익/수익률, 모두 콤마 숫자+원), **연도별 SVG 성장 그래프**
(원금 vs 평가금액 vs 누적배당), **동기부여 인사이트**(마지막 해 월 배당액, 목표 N억 도달 시점),
**연도별 상세 표**. **시장 시나리오** 프리셋(보수/중립/공격)은 주가상승률·배당률·배당성장률·
배당주기만 바꾸고 **초기원금·월적립·기간(사용자 상황)은 절대 변경 안 함**. 시나리오 값은
과거 데이터 기반(코스피 배당주 장기 총수익 ~8%, 밸류업 배당증가, SCHD 배당성장 ~11% 참고로
한국에 보수 적용): 보수 주가2%/배당4.5%/성장3%, 중립 4%/4.5%/5%, 공격(리밸런싱 성공 가정) 6.5%/5%/7%.
→ Claude Code 확장 아이디어: 점수 상위 N종목 자동 포트폴리오 연동, 실제 주가 기반 백테스트.

### 6-1. 내 종목 적용 (구현 완료, `addToPortfolio`/`setQty`/`renderPortfolio`)
계산기 상단에서 대시보드 종목을 골라 담고 **종목별 보유 수량(주)을 입력**하면:
- 종목별 평가금액(현재가×수량) 합계가 **초기 원금에 자동 반영**
- **금액가중 평균 배당률**(비중 큰 종목에 가중)이 계산기 배당률에 반영
- 현재 배당률 vs 4년평균(금액가중)으로 **배당성장률 자동 추정**(0~10% 제한)
주가상승률은 시나리오 버튼으로 별도 조절. (과거 가정 모드는 제거됨 — 미래 예측 단일.)
→ Claude Code 확장: 실시간 크롤링 현재가와 연동, 평균매입가 입력받아 손익까지 표시.

## 7. 내 포트폴리오 페이지 (구현 완료, lot 기반)
**종목 추가는 검색 방식**(종목명·업종 입력 → 후보 클릭). 계산기 '내 종목'도 동일 검색 UI.
**분할매수 누적(방식 A)**: 종목별 `lots:[{qty,price,date}]` 배열에 매수 건을 쌓고 총수량·평단(가중평균)
자동 계산. 표는 종목별 요약 한 줄(▸ 클릭 시 펼쳐 개별 매수내역 표시·삭제, '추가 매수' 버튼).
**매입가에 '현재가로' 체크박스** — 체크 시 현재가 자동 입력. 입력: 수량+매입가(or 현재가)+매입일.
표시·리밸런싱 신호·차트·계산기 공유는 기존과 동일(`rollup()`으로 평가). 데모는 메모리 저장.
→ Claude Code: 백엔드 holdings(lots 포함) 영구 저장, 실시간 현재가 연동, 매도 기록·실현손익까지 확장.

## 7. 주의·한계 (사용자에게 이미 고지됨)
- 과거 PER/PBR 10년 밴드는 원본 엑셀에 없어 미구현. 필요 시 유료 데이터로 별도 보강.
- 크롤링은 개인 참고용. 상업적 재배포 금지. 요청 과다 시 차단 → 캐시(30분)+딜레이 적용됨.
- 점수는 상대평가이며 미래 수익 보장 아님.
